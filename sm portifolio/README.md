# Hayat Khayredin – Digital Marketer Portfolio

Static portfolio site (HTML, CSS, JavaScript).

## Live site (GitHub Pages)

After you turn on Pages (see below), your site will be available at:

**https://hayatkheyredin.github.io/my-portfolio/**

Anyone with that link can open it in a browser—no login required.

### Enable GitHub Pages (one-time)

1. Open your repo on GitHub: **[github.com/Hayatkheyredin/my-portfolio](https://github.com/Hayatkheyredin/my-portfolio)**
2. Go to **Settings** (repo menu).
3. In the left sidebar, click **Pages** (under “Code and automation”).
4. Under **Build and deployment** → **Source**, choose **Deploy from a branch**.
5. **Branch:** `main` → **Folder:** `/ (root)` → **Save**.
6. Wait 1–3 minutes. Refresh the **Pages** section; you’ll see “Your site is live at …”.

### Update the live site after edits

From this folder in a terminal:

```bash
git add .
git commit -m "Describe your change"
git push origin main
```

GitHub Pages will rebuild automatically (usually within a minute).

### Preview on your computer

Open `index.html` in your browser, or use a simple local server if you prefer.

## Tips

- **Custom domain** (optional): In **Settings → Pages**, add your domain and follow GitHub’s DNS instructions.  
- **Repo visibility**: Public repos work with free GitHub Pages; private repos need a paid plan for Pages on some accounts—check [GitHub’s docs](https://docs.github.com/pages).
